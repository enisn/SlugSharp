# Security Policy

## Reporting a Vulnerability

We take the security of this repository seriously. If you discover a security vulnerability, please follow these steps to report it to us:

- Email us at [furkan.dvlp@gmail.com](mailto:furkan.dvlp@gmail.com) with a detailed description of the vulnerability.
- Provide steps to reproduce the vulnerability, along with any relevant code or configuration.
- We will acknowledge your email within [20 business days] and work with you to address the issue.

## Supported Versions

We strive to provide security updates for the current and previous major versions of this repository. It is recommended to always use the latest stable version to ensure you have the latest security patches.

## Security Updates

- We are committed to addressing security vulnerabilities and will work to provide updates in a timely manner.
- Security updates will be released as new versions of the repository. It is your responsibility to update your project to the latest version to benefit from these updates.

## Reporting Other Issues

For general issues, bugs, or feature requests unrelated to security, please use the GitHub Issues section of this repository. We appreciate your contributions to improving this project.

## Security Best Practices

- Keep your dependencies up to date to benefit from the latest security patches.
- Follow secure coding practices, such as input validation and output encoding, to mitigate common security vulnerabilities.
- Regularly review and audit your code for potential security issues.
- Secure your development and production environments by following best practices for server and network security.

## Scope

This Security Policy applies to this repository and its maintainers. It does not cover vulnerabilities or security issues arising from the use of this repository in combination with other software or libraries.

## Attribution

We appreciate and acknowledge the responsible disclosure of security vulnerabilities by individuals or organizations. Upon request, we can provide recognition for such contributions in our release notes or other appropriate means.
